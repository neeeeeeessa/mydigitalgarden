    HTML (documentation)

    Doctype

  • Use <!DOCTYPE html> at the top of the page to indicate to the browser that this is HTML5. If not used, will most likely not change anything, but might cause issues sometimes.

      • HTML

    • Use <html> & </html> to indicate the following code is HTML.

        • Head

      • Use <head> & </head> to include:
        • <title> Title placeholder </title> | add page name on the tab.
        • <link rel="stylesheet" href="styles.css"/> & <link href="https://fonts.googleapis.com/..." rel="stylesheet"> | add css links.
        • <script src="index.js"> & </script> | add js links.
        • [tbd] Metadata for sharing in other websites (e.g. Twitter).

        Body

      • Use <body> & </body> to include page elements:

          • Text elements (Core HTML tags)

        • <h1> & </h1> | add title type 1.
        • <h2> & </h2> | add title type 2.
        • <h3> & </h3> | add title type 3.
        • <p> & </p> | add paragraph.

          • Anchor Tags

        • <a href="file.html"> Visible Text </a> | add a link to another html file.
        • <a href="website.com" target="_blank"> Visible Text </a> | add a link for an external website to open on a new tab.

          • Images

        • <img src=""> | includes image. The image must be placed in the same folder as the coding file.
        • <img src="" width="100%> | includes image and width. Adding width="100%" resizes the image according to page size.
        • <img src="" alt="image description"> | includes image and alt text for image description. The alt text does not show in the rendered file, but is used for accessibility purposes when users use screen readers to access the file. The alt text also works as a fallback if the image doesn't render properly, and it helps your file to get more search traffic (e.g. on Google) since it provides metadata which can be used by other tools. Best practices for writing alt text include:
          • Descriptions should remain under 125 characters: many screen readers will cut off after this limit has been reached.
          • Do not include "image of": alt text is within the image tag already.

          Interactive Elements & Buttons

        • <button> & </button> | add pre-formated buttons.
        • <input type="text"> | add input box. Add placeholder="" inside <> to include placeholder text.
        • <input type="password"> add input password box.
        • <input type="checkbox"> add input checkbox.
        • <label for="name">Title outside of button: </label>
          <input placeholder="placeholder inside of button" id="name"> | add labels. Labels ensure that screenreaders can convey the purpose of the input field for accessibility and allows users to click on the label to focus or activate the associated input. Use for and id where the value is the same for both.
        • <time datetime="2025-05-13">May 13, 2025</time> | add time. It includes a datetime attribute for machine readable format, allowing for better search engine results.
        • <address> & </address> | add address.
        • <audio controls><source src="file.wav" type="audio/mpeg"></audio> | add audio.

          • Section Elements

        • <div> & </div> | add section.
        • <br> & </br> | add break.

          • List Elements

        • Use <ol> & </ol> and <ul> & </ul> to start ordered and unordered lists.
          • <dl> & </dl> | add description list.
            • <dt> & </dt> | add description title.
            • <dd> & </dd> | add description detail.
            • <li> & </li> | add list item.

          HTML File Paths

          Adding "./file" means that file is in the same folder as the html file where it's referenced; adding "/file" means that file is at the root of the project; Adding "../file" means that file is in a folder up in relation to the html file where it's referenced.

          • <img src="picture.jpg"> | The "picture.jpg" file is located in the same folder as the current page.
          • <img src="images/picture.jpg"> | The "picture.jpg" file is located in the images folder in the current folder.
          • <img src="/images/picture.jpg"> | The "picture.jpg" file is located in the images folder at the root of the project. Having a slash is a way to start directly from the root directory.
          • <img src="./picture.jpg"> | The "picture.jpg" file is located in the current folder. Adding "./" does not change anything, it's just a visual reminder.
          • <img src="../picture.jpg"> | The "picture.jpg" file is located in the folder one level up from the current folder.
          • <img src="../../picture.jpg"> | The "picture.jpg" file is located in the folder two levels up from the current folder.

    CSS

    CSS selects certain HTML elements and styles them (e.g. body, button, img). Within brackets, a property and a value are required as in the example below:
    body {
     color: red;
    }

    CSS Syntax

  • background: red (or #8B4513); | adds background colour.
  • color: white (or #8B4513); | adds letter colour.
  • text-align: center; | aligns text.
  • width: 50px; | adds width and is not needed for buttons, because the element already has built-in width.
  • height: 36px; | adds height.
  • font-size: 25px; | adds font size.
  • font-weight: bold; | adds font formatting.
  • font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif; | adds font type by targetting a font stack: in the example, if the browser finds the Verdana font installed, it will use; else it will successfully try the next ones in the family. and are generally placed directly in the body instead of individual elements. Web safe fonts are fonts that are available in most clients (e.g. laptops, smartphones, tablets, smartwatches, etc.). See CSS Web Safe Fonts for a list of the best web safe fonts.
  • text-align: center; | adds text alignment.
  • border: 1px solid #dfe1e5; | adds border and requires 3 values: size, type and color.
  • border-radius: 4px; | rounds border corners.
  • margin-top: 30px; | adds margin on top.
  • margin: 10px; | adds margin to all sides.
  • padding-left: 3px; | adds padding on the left.
  • padding: 5px; | adds padding to all sides.
  • line-height: 24px; | adds margin: it refers to the height of one line of text and is better to use with input fields as more browsers display it correctly. It does not take into account padding and border values.
  • box-shadow: 10px 10px 5px lightblue; | adds shadow by specifying horizontal shadow size, vertical shadow size, blur effect and shadow color. For additional elements, see Box Shadow.

    • Inline & Block Elements

    Display inline stacks elements side to side, meanwhile display block adds a block on the side of each element to ocupy the empty space, resulting in vertical stacking of elements.

    Classes

    Classes are used to identify elements in html, which will be referred to in css.

  • <div class="class_name"> & </div> are used to identify different div elements (e.g. class="main").
    ".class_name" is used in CSS to identify the right div (e.g. .main).
  • Margin-top in html is commonly replaced by <div> & </div> in CSS because it is easier to control the distances between elements.

    • Utility Classes

    Utility classes are classes set for a single CSS property, and can be used for multiple elements by renaming individual elements and adding the class name after a single space:
    In HTML: <h3 class="italic">Heading</h3> or <h3 class="card italic">Heading</h3>
    In CSS: .italic {
     font-style: italic;
    }

    Margins

    Margins add space on the outside of a div. By default, a margin of 8px is always applied:
    body {
     margin: 8px;
    }

  • The DRY approach to adding margins with different values is by using "margin: top right bottom left":
    .body {
     margin: 10px 20px 30px 40px;
    }
  • In case there are repeated values for top-bottom (e.g. 10px) and/or for left-right (e.g. auto), they can be combined:
    .body {
     margin: 10px auto;
    }
  • Margins do not apply to bottom because the browser can be extended.
  • Margin-top and margin-botton will in many cases collapse into each other, and the biggest value will prevail (although there are scenarios that this doesn't happen).

    • Centering

    There are 3 things that must be in place when centering with margins:

  • display: block; | adds space next to the element.
  • width: ?px; | sets the element's size.
  • margin-left: auto; & margin-right: auto; | makes the element take all available space.

    • Alternatively, centering can also be done by removing display and margin configurations, and adding "text-align: center;" to .main in css.

    Padding

  • Refers to the distance between the background of an element and the inner elements, or between the edge and the div container (e.g. background and text). In PowerPoint would be the text distance configuration in Format Shape.
  • By default the padding of a div is 0.

    • The Box Model

  • Refers to creating boxes by adding margin, border, padding and content to an element.
  • Context colors are free online websites to check the contrasting ration of colors for accessibility (e.g. userway.org/contrast).

    • Flexbox

  • Flexbox (short for Flexible Box Layout) is a CSS layout model that provides a more efficient way to lay out, align, and distribute space among items in a container, whether they are arranged in rows or columns. It's particularly useful for responsive design, allowing elements to dynamically adjust their size to fill available space or shrink to fit, eliminating the need for complex positioning and floats. Flexboxes consist of:
    • A container: div
    • Its direct children: button, input, etc.

  • To display items horizontally (in a row), create a flexbox by adding a div and a class in html, and add "display: flex;" inside the related class in the css file.
    <div class="btn-wrapper">
     <button class="btn">Google Search</button>
     <button class="btn">I'm Feeling Lucky</button>
    </div>

    .btn-wrapper {
     display: flex;
    }

  • When centralizing more than one element in the same row, use margin-left and margin-right inside the div class to add spacing between the elements inside div.
    .btn {
     margin-left: 4px;
     margin-right: 4px;
    }

  • To centralize the items using flexbox, use "justify-content: center" or "justify-content: space-around" or "justify-content: space-between" inside the related class in the css file.

    • Example 1:

    .btn-wrapper {
     display: flex;
     justify-content: center;
    }

    Example 2:

    .card {
     display: flex; | creates flexbox.
     justify-content: space-around; | centralizes elements inside the card, which can also be accomplised by adding "width: fit-content;" or adding "margin-right: ?px;" inside the element {} on the left of the div.
     width: 400px; | centralizes the card on the page.
     margin-left: auto; | centralizes the card on the page.
     margin-right: auto; | centralizes the card on the page.
    }

    CSS Inheritance

    Referes to adding a parameter to the parent which will be inherited by all of the children of the element. Inheritance doesn't just work with the direct children of the element selected in CSS, but will also work any number of layers down the descendant tree of the selected element. However, not all CSS properties get inherited; see Inherited CSS Properties for a list of inherited css properties. It helps you write DRY (Don't Repeat Yourself) code, and allows you to avoid bugs and be more efficient.
    .article {
     text-align: center;
    }
